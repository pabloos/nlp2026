6.1 El Ecosistema de LLMs¶

Fine-tuning vs Uso de API¶

Estrategia Ventajas Cuándo usarla
API (zero-shot/few-shot) Sin entrenamiento, rápido, flexible Prototipado, tareas generales
Fine-tuning Control total, privacidad, menor latencia Dominio muy específico, escala
RAG Actualizable sin re-entrenar Información externa o muy reciente
Quantización + local Privacidad, coste cero Datos sensibles, sin internet

6.2 Fundamentos de Prompt Engineering¶

Zero-shot, One-shot y Few-shot¶

==================================================
  Zero-shot
==================================================

Classify the sentiment of this review as POSITIVE or NEGATIVE.
Review: "The battery died after 2 hours."
Sentiment:
  → Respuesta esperada: NEGATIVE

==================================================
  One-shot
==================================================

Classify the sentiment of product reviews.

Review: "Amazing sound quality!"  → POSITIVE

Review: "The battery died after 2 hours."
Sentiment:
  → Respuesta esperada: NEGATIVE

==================================================
  Few-shot
==================================================

Classify the sentiment of product reviews.

Review: "Amazing sound quality!"          → POSITIVE
Review: "Broke after one week."            → NEGATIVE
Review: "Worth every penny, love it."     → POSITIVE
Review: "Returned it immediately."         → NEGATIVE

Review: "The battery died after 2 hours."
Sentiment:
  → Respuesta esperada: NEGATIVE

Instrucciones de rol y formato de salida¶

PROMPT:

SYSTEM:
You are a product review analyst. Extract structured information from reviews.
Always respond with valid JSON only, no additional text.

USER:
Analyze this review:
"The Sony WH-1000XM5 headphones are amazing! The noise cancellation is top-notch 
and the battery lasts 30 hours. A bit expensive at $399 but totally worth it."

Respond with this JSON schema:
{
  "product": "...",
  "sentiment": "positive|negative|neutral",
  "aspects": {
    "aspect_name": {"sentiment": "...", "mention": "..."}
  },
  "price_mentioned": "...",
  "rating_estimate": 1-5
}

RESPUESTA ESPERADA:

{
  "product": "Sony WH-1000XM5",
  "sentiment": "positive",
  "aspects": {
    "noise_cancellation": {"sentiment": "positive", "mention": "top-notch"},
    "battery":             {"sentiment": "positive", "mention": "lasts 30 hours"},
    "price":               {"sentiment": "mixed",    "mention": "expensive at $399"}
  },
  "price_mentioned": "$399",
  "rating_estimate": 5
}

Etiquetado Zero-shot a Gran Escala¶

Ver etiquetado.py — facebook/bart-large-mnli etiqueta aspectos sin datos de entrenamiento.

# 1. Modelo: facebook/bart-large-mnli
# 2. Sin datos de entrenamiento → etiqueta directamente

from transformers import pipeline
zsc = pipeline("zero-shot-classification", model="facebook/bart-large-mnli")

aspect_labels = ["battery life", "sound quality", "build quality",
                 "price and value", "connectivity", "comfort"]

review = "The noise cancellation is incredible and earbuds are super comfortable."
result = zsc(review, candidate_labels=aspect_labels)
# → {'labels': ['sound quality', 'comfort', 'build quality', ...],
#    'scores': [0.72, 0.63, 0.41, ...]}

# Filtrar por confianza
HIGH_CONF = 0.5
high_conf_labels = [l for l, s in zip(result["labels"], result["scores"]) if s > HIGH_CONF]

RAG — Retrieval-Augmented Generation¶

Ver rag.py — chatbot de reseñas Amazon con FAISS + Sentence Transformers.

# ── FASE OFFLINE: indexar documentos ──────────────────────────────────────
from sentence_transformers import SentenceTransformer
import faiss, numpy as np

embedder = SentenceTransformer("all-MiniLM-L6-v2")  # 23M params, rápido
docs = [review["text"] for review in amazon_reviews]  # corpus de reseñas

embeddings = embedder.encode(docs, show_progress_bar=True)  # (N, 384)
index = faiss.IndexFlatIP(384)  # similitud coseno aproximada
index.add(embeddings.astype("float32"))

# ── FASE ONLINE: responder preguntas ──────────────────────────────────────
query = "What do people say about the battery life?"
q_emb = embedder.encode([query]).astype("float32")

_, indices = index.search(q_emb, k=5)          # top-5 más similares
contexto = "
".join([docs[i] for i in indices[0]])

prompt = f"""Answer the question based ONLY on the following reviews:

{contexto}

Question: {query}
Answer:"""
